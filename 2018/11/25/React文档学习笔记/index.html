Introducing JSX

What is JSX

  • a synctax extension to JavaScript

  • Use it with React to describe what the UI should look like

  • JSX produces React “elements”

Why JSX

  • binding rendering logic with other UI logic: how events are handled, how the state changes over time. and how the data is prepared for display
  • React seperate concerns into ‘Components’, as JSX works as a technique to aid acquiring such goal

Embedding Expressions in JSX

  • Declare variables in JSX by wrapping them in curly braces

  • Any valid JavaScript expression can be put inside the curly braces in JSX

    • i.e. 1+1, user.firstName, formatName(user)

Specifying Attributes with JSX

  • JSX can be used by wrapping curly braces and embeded in JavaScript expressions in an attribute

  • Don’t put quotes around curly braces when embedding a JS expression in an attribute

  • quotes and curly brances can be used one in a time, cannot use both in the same attribute

  • use camelCase property naming convention inside of HTML attribute names

JSX Prevents Injection Attracks

  • By default, React DOM escapes any values embedded in JSX before rendering them
  • everything is converted to a string before being rendered

小结

  1. JSX本质上是一种JS的语法，它帮助react用JS的形式来定义页面上的UI如何呈现
  2. React用到了separation of concerns的思想，把项目细化到Components组件的层面，JSX帮助组件来实现其中的UI
  3. JSX是一种JS语法，因此也可以在里面插入JS的表达式，方式是使用大括号
  4. 不仅可以在元素内容里插入JS表达式，也可以在元素的属性中使用JS表达式。需要注意的是，如果使用了JS表达式，就不能在表达式外使用引号；引号和大括号只能使用一个
  5. JSX虽然看起了像HTML，但实际上是JS，所以属性命名沿用JS的驼峰命名法
  6. JSX会防止注入攻击，所以所有渲染的内容都会先转译成字符串
  7. JSX实际上通过Babel的解析，通过React.createElement()方法将元素内容转译成了一个对象，对象内容包括元素的type, className和children

Rendering Elements

  • Unlike browser DOM elements, React elements are plain objects, and are cheap to create

  • Normally, application built with one React root DOM node, but it can have as many isolated root DOM nodes as you like

  • React elements are immutable, once you create an element, you can’t change its children or attributes

  • most React apps only call ReactDom.render() once

  • React DOM compares the element and its children to the previous one, and only applies the DOM updates necessary to bring the DOM to the desired state

小结

  1. react的元素是组成组件的基础
  2. 通常react应用只有一个DOM根节点，但是如果你在已有的应用里引入react，也可以有多个独立的根节点
  3. 通常react应用都只调用ReactDOM.render()方法一次
  4. react更新组件内容时，会和之前的组件元素比较，只更新有变化的部分

Components and Props

  • Conceptually, components are like JS functions that accepets arbitrary inputs (called “props”) and return React elements describing what should apprear on the screen

Function and Class Components

JS function

  • simplest way to define a component

  • it’s called function components

  • it accepts a single object argument and returns a React element

ES6 class

These two components are equivalent from React’s point of view

Rendering a Component

  • elements can represent user-defined components

  • 当出现自定义组件时，react会传一个JSX attributes给组件，这个JSX attribtues就是props

  • Always start component names with a capital letter, because React treats components starting with lowercase letters as DOM tags

Extracting components

  • better naming props from the component’s own point of view rather than the context in which it is being used
  • if a part of your UI is used several times, or is complex enough on its own, it is a good candidate to be a reusable component

Props are Read-Only

  • never modify components’ own props

Pure functions VS impure functions

  • Pure functions do not attempt to change their inputs, and always return the same reuslt for the same inputs

  • impure functions change its own input

  • All react components mush act like pure functions with respect to their props

小结

  • 组件的写法有两种：JS函数和class方法

  • 会重复使用到的部分应该独立为单独的组件

  • 组件中的属性会通过props由父组件传递给它
  • props是只读的，任何时候都不应该修改props的值

State and Lifecycle

  • state is private and fully controlled by the component

Lifecycle

  • mounting: when a component is rendered to the DOM for the first time, along with componentDidMount()method
  • unmounting: when the DOM produced by the component is removed, along with componentWillUnmount() method
  • These methods are called lifecycle methods

state

  • this.state has a special meaning, you are free to add additional fileds to the class manually if you need to store something that doesn’t participate in the data flow

Something You Should Know About State

  • Do not modify state directly, use setState() instead. The only place you can assign this.state is the constructor

  • state updates may be asynchronous

    • react may batch multiple setState() calls into a single update for performance

    • you should not rely on their values for calculating the next state

    • to fix it, use a second form of setState() which accepts a function rather than an object

  • state updates are merged

    • the merging is shallow, only merge the part that has declared new value

The data flows down

  • neither parent or child components can know if a certain component is stateful or stateless
  • a component may choose to pass its state down as props to its child components

小结

  • 每个元素都有自己的生命周期以及对应的方法
  • state用来存储UI上会变化的值
  • state只能通过setState()方法来改变，而不应该通过赋值的方式来改变
  • setState()方法不是立即触发的，它是异步的
  • state被修改时，是通过合并的方式得到新的state，即传入的部分即修改的部分，未传入的值会保持原有state的值

Handling Events

handling events with React elements is similar to doing it on DOM elements:

  • React events are named using camelCase

  • With JSX, you pass a function as the event handler rather than a string

  • you cannot return false to prevent default behaviour in React, you must call preventDefault explicitly

this in event listener

  • you have to be careful about the meaning of this in JSX callbacks, because in JS, class methods are not bound by default

  • Generally, if you refer to a method withou () after it, you should bind that method

  • If you are annoyed by calling bind, there are two ways to get around this:

    • use experimental public class fields syntax 即在class内部，非render内部，使用箭头函数的方式来绑定this

    • use arrow function in the callback 在调用函数的时候使用箭头函数

      • the problem with this syntax is that a different callback is created each time LoggingButton renders

小结

  • JSX在使用事件监听器时，应使用变量的方式，而不是字符串的方式
  • JSX调用的事件监听无法通过return false来防止默认事件，应直接指明preventDefault()
  • 调用事件监听函数时需要注意this的指向，需要早constructor里给函数绑定this
  • 除此之外，也可以通过在class内部使用箭头函数来声明函数的方式绑定this，以及在调用函数的时候使用箭头函数来绑定this
  • 给事件监听函数传参时，可以通过箭头函数来传参，也可以通过bind方法来绑定

Conditional Rendering

  • if statement to decide which element should be rendered
  • Make conditional rendering inline with local operators like &&, xx ? xx : xx
  • whenever conditions become too complex, it might be a good time to extract a component

hide a component

  • when you want a component to hide itself, return null instead of its render output
  • returning null from a component’s render method does not affect the firing of the component’s lifecycle methods, for instance componentDidUpdate will still be called

小结

  • 可以通过JS中的条件判断或者逻辑运算符来决定元素是否需要被渲染
  • 当判断逻辑过于复杂时，应考虑将这部分元素独立成一个组件
  • 如果一个组件需要被隐藏，则返回null即可
  • 当组件返回null时，它的声明周期方法并不受影响

Lists and Keys

  • use map to iterate
  • assign a key to all items
  • keys help React identify which item have changed, added, or removed
  • the best way to pick a key is to use a string that uniquely identifies a list item among its siblings
  • using index for keys is not recommended since the order of items may change
  • If an explicit key is not assigned to list items, React will default to using indexes as keys
  • keys only make sense in the context of the surrounding array
  • a good rule of thumb is that elements inside the map() call need keys
  • keys only need to be unique among siblings, they don’t need to be globally unique
  • keys serve as a hint to React but they don’t pass to your components 也就是说不可以当做id来用

小结

  • 通过map方法遍历数组来生成一组元素
  • 每个元素都应有一个独特的key，这个key用于与兄弟元素区别开，因此在兄弟元素中独特即可，不用在全局范围内独一无二，且key不会被用作id
  • 这个key用来表示哪个元素发生了改变，因此不建议使用索引来作为key，因为数组的顺序可能会改变

Forms

Controlled Components

  • With a controlled component ,every state mutation will have an associated handler function
  • you can set the initial value of the input, textarea by setting the value in the state in constructor. so that the text area starts off with some text in it

Handling multiple inputs

  • add name attribute to each element and let the handler function choose what to do based on the value of event.target.name

Uncontrolled component

  • its value is read-only

Lifting State Up

  • in React, sharing state is accomplished by moving it up to the closest common ancestor of the components that need it. It is called lifting state up
  • Name the value and the event handler as value and onChange is a common convention
  • there should be a single “source of truth” for any data that changes in a React application
  • if something can be derived from either props or state, it probably shouldn’t be in the state

Composition vs Inheritance

Containment

  • Some components don’t know their children ahead of time

  • This is especially common for components like Sidebar or Dialog that represent generic “boxes”

  • in such condition, we recommend use the special children prop to pass children elements directly into their output

Thinking in React

Break the UI into a component hierarchy

  • draw boxes around every component accroding to single responsibility principle

Build a static version in react

  • state is left for interactivity. for immutable values, use props instead

Identify the minimal representation of UI state

  • what kind of value should be stored in state
    • is it passed in from a parent via props
    • does it remain unchanged over time
    • can you compute it based on any other state or props in your component

Identify where your state should live

  • identify every component that renders something based on that state
  • find a common owner component
  • either the common owner or another component higher up in the hierarchy should own the state
  • if you can’t find a component where it makes sense to own the state, create a new component simply for holding the state and add it somewhere in the hierarchy above the common owner component
